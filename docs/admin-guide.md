# Using HTML Cache

This guide is for owners and operators who manage page caching. Every step uses the labels you see on screen.

## What you're looking at

| Screen or signal | What it tells you |
| --- | --- |
| **Cache coverage** | How many of your page URLs are cached |
| **Cache map** | Which URLs are cached and what content they depend on |
| **Cached URLs** | The specific pages currently served from cache |

## What to do when...

| Situation | What to do |
| --- | --- |
| A change isn't showing on the live site | Clear the cache for that page (use **Clear selection**), or clear all |
| You're about to make big updates | Turn on **maintenance mode** first |
| Coverage looks low | Some pages aren't cached yet; they will cache as visitors load them |
| A page depends on changed content | Use the **Cache map** dependencies to clear the right URLs |

## Settings and retention

- Cached copies refresh automatically as content changes, but you can always clear manually.
- Use maintenance mode only while updating, and turn it off when done so visitors see the site again.
